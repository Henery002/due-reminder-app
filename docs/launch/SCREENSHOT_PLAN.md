# 到期提醒助手应用市场截图清单

> 创建日期：2026-05-05  
> 当前状态：截图脚本前的场景规划；实际尺寸、数量和机型需按渠道复核。

## 截图叙事顺序

### 1. 首页看清最近要处理什么

1. 页面：首页
2. 画面重点：首页总览卡片、近期事项、分类胶囊
3. 截图文案：最近要处理什么，一眼就知道
4. 数据建议：今日 1 件、近 7 天 3 件、逾期 0 件

### 2. 30 秒添加一个到期事项

1. 页面：新建事项
2. 画面重点：类型选择、常用模板、日期选择
3. 截图文案：会员、账单、证件，到期日快速记录
4. 数据建议：视频会员 / 信用卡年费 / 驾驶证换证

### 3. 提醒方式自己决定

1. 页面：新建事项 / 全部事项
2. 画面重点：“安排本地提醒”开关、提醒计划摘要、全部页“提醒方式”筛选
3. 截图文案：需要提醒就打开，只想留档也可以
4. 数据建议：一个本地提醒事项、一个仅记录事项
5. 注意：不要承诺 100% 触达，不把仅记录说成系统通知。

### 4. 延后、已处理、编辑都很快

1. 页面：首页或全部事项
2. 画面重点：事项卡片操作、延后底部面板
3. 截图文案：处理不了，就延后到更合适的时间
4. 数据建议：水电缴费、网盘续费、护照到期

### 5. 本地备份更安心

1. 页面：数据备份页
2. 画面重点：导出备份文本、导入备份
3. 截图文案：不急着上云，也能先把数据握在手里
4. 注意：展示时避免露出真实私人数据。

## 截图准备事项

1. 准备一套干净演示数据，避免真实个人事项。
2. 统一使用浅色模式。
3. 截图前确认状态栏时间、电量和网络图标不抢戏。
4. 国内应用市场尺寸要求差异较大，导出前按目标市场逐一复核。
5. 若后续调整视觉主题或图标，需同步更新截图。

## 演示数据入口

当前可在 App 内进入：

1. `我的`
2. `数据备份`
3. 点击 `生成截图演示数据`

该入口会写入一组以 `demo-` 开头的安全示例事项，覆盖同 ID 演示数据，不会清空其他用户数据。正式截图前仍建议使用干净测试环境，避免混入真实个人事项。
