# 提醒策略产品规则

> 最后更新：2026-05-06  
> 当前阶段：M3.7 表单与列表体验继续精修  
> 适用范围：本地事项、提醒模式、提醒点、延后、备份兼容和页面说明口径。

## 目标

提醒策略的核心目标是让用户明确知道一件事会不会触发系统通知，并能在轻量表单内完成常用配置。

当前版本优先保证以下体验：

1. 默认创建本地提醒模式的事项。
2. 支持只保存记录，不向系统安排通知。
3. 支持按事项类型生成合理默认提醒点。
4. 支持少量自定义提醒点，但避免通知过密。
5. 所有提醒模式必须能被列表、编辑、延后和备份正确识别。

## 提醒模式

事项使用显式字段 `reminderMode` 表达提醒方式。

| 模式 | 展示文案 | 产品含义 | 通知行为 |
| --- | --- | --- | --- |
| `notify` | 本地提醒 | 保存事项，并向系统安排本地通知 | 允许生成、取消、重排本地通知 |
| `record-only` | 仅记录 | 只保存到期记录，不向系统安排通知 | 不安排新通知，编辑旧事项关闭提醒时会取消旧通知 |

规则：

1. 新建事项默认使用 `notify`。
2. 用户可在新建/编辑页关闭“安排本地提醒”，将事项保存为 `record-only`。
3. 列表卡片对 `record-only` 显示“仅记录”标签；`notify` 不额外展示模式标签。
4. 全部事项页的“提醒方式”筛选独立于状态筛选，选项为“全部方式 / 本地提醒 / 仅记录”。
5. 不使用空 `reminderRules` 反推用户意图，后续扩展仍应依赖 `reminderMode` 这类显式字段。

## 默认提醒点

默认提醒点由事项类型决定：

| 类型 | 默认提醒点 |
| --- | --- |
| 订阅 | 提前 7 天、提前 1 天、到期当天 |
| 账单 | 提前 3 天、提前 1 天、到期当天 |
| 证件 | 提前 60 天、提前 30 天、提前 7 天 |

规则：

1. 默认提醒时间为提醒日期当天 09:00。
2. 已经过期的提醒点不会展示成可安排提醒，也不会安排为未来通知。
3. 如果到期日已经没有可安排的未来提醒，表单可保存为记录，但不会自动触发新提醒。
4. 切换事项类型时，表单会重置为该类型的默认提醒点。

## 自定义提醒点

用户可在新建/编辑页为单个事项添加自定义提前天数。

规则：

1. 自定义提醒点允许范围为 0-365 天。
2. 输入必须是整数，不接受空输入、负数、小数或非数字内容。
3. 单个事项最多保留 5 个提醒点，包含默认提醒点和自定义提醒点。
4. 重复提醒点不重复加入计划。
5. 提醒点按提前天数从大到小排序生成规则。
6. 保存后，自定义提醒点必须和默认提醒点一起参与通知生成、取消和重排。

## 保存与编辑

新建事项：

1. `notify` 模式：保存前展示将安排的未来提醒次数，保存时尝试申请通知权限并安排本地通知。
2. `record-only` 模式：保存前展示“将保存为仅记录，不发送本地通知”，保存时不安排本地通知。

编辑事项：

1. 保存编辑会先取消旧通知，再按新规则处理。
2. 从 `notify` 改为 `record-only` 时，会取消旧通知，并将 `reminderRules` 清空。
3. 从 `record-only` 改为 `notify` 时，会按当前到期日和提醒点重新生成未来提醒。
4. 编辑已处理事项时保持已处理状态，不因编辑自动恢复为未处理，也不重新安排本地通知。

## 延后与完成

规则：

1. `notify` 事项可展示“延后”操作，并可重排延后通知。
2. `record-only` 事项不展示“延后”操作，也不会通过延后重新调度通知。
3. 点击“已处理”会取消该事项已安排的本地通知，并写入完成时间。
4. 删除事项会取消该事项已安排的本地通知，并移除本地记录。

## 备份与兼容

规则：

1. 备份导出会保留 `reminderMode` 和 `reminderRules`。
2. 旧备份如果缺少 `reminderMode`，导入时默认兼容为 `notify`。
3. SQLite 旧库通过迁移补充 `reminderMode TEXT NOT NULL DEFAULT 'notify'`。
4. 备份仍是本地 JSON 文本能力，不代表账号、云同步或跨设备自动同步已经上线。

## 页面说明口径

当前版本必须保持以下解释口径：

1. 说“向系统安排本地提醒”，不承诺“永久不漏提醒”。
2. 明确拒绝通知权限不影响记录事项，只影响系统通知触达。
3. “仅记录”适合补录已过期事项、线下已处理但想留档的事项，或暂时不想收到系统通知的事项。
4. Pro、云同步、账号、广告、OCR 和真实支付仍是暂缓能力，不要在提醒策略说明中暗示已经上线。

## 暂缓事项

以下能力暂不纳入当前提醒策略：

1. 云端同步提醒规则。
2. 账号级多设备提醒。
3. Pro 级多级提醒模板。
4. OCR / AI 自动识别到期日。
5. 系统级后台保活或厂商通道保障。
6. 真机长时间、锁屏、跨日通知专项复核。
