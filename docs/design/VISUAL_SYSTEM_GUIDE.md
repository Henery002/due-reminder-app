# 到期提醒助手视觉系统规范

> 创建日期：2026-05-05  
> 当前状态：第二版整改规范，后续 UI 迭代必须遵循。
> 2026-05-06 补充：中国大陆市场 UI 审美适配要求，避免过度欧美化视觉表达。

## 1. 现状判断

当前“像大字版”的感受是合理的，不是单纯主观偏好。

主要原因：

1. 多处页面标题使用 `28-30` 号字，且叠加 `800/900` 字重。
2. 卡片标题、按钮、标签、角标大量使用 `900`，导致信息层级都在抢焦点。
3. 页面外边距普遍 `20`，卡片内边距多为 `15-20`，在手机屏幕上显得厚重。
4. 胶囊按钮和行动按钮高度偏大，视觉更接近无障碍大字号模式。
5. `我的` 页分区虽然清楚，但 hero、section title、row title、chevron 都偏粗，造成“全页面都很用力”的感觉。

这类到期提醒工具应更接近“清爽生活效率 App”，而不是老式工具箱。年轻化不等于所有元素更大、更圆、更粗；更重要的是留白节奏、轻重对比、微动效和色彩克制。

## 2. 参考方向

本项目参考方向：

1. 国内主流移动端设计体系的共同点：确定、自然、轻量、模块化，优先降低用户认知成本。
2. 国内年轻化生活工具的常见做法：页面标题不必过大，设置页更偏列表聚合，入口行更轻，色彩用于状态和焦点，不用于铺满所有卡片。
3. Ant Design Mobile 一类移动组件体系：强调高性能、可配置、原子化组件、顺滑手势和精致动效，适合作为本项目组件化和主题化参考。
4. Material Design 3 只作为字号层级、色彩角色和无障碍边界参考，不作为本项目的视觉主风格。
5. Todoist、Microsoft To Do 一类海外效率应用只参考信息克制与任务优先，不直接照搬大留白、重插画、强品牌色块或欧美 SaaS 式 hero 结构。

## 3. 中国大陆市场 UI 审美适配

本项目面向中国大陆 Android 用户，视觉风格应更接近“清爽生活工具 + 国内移动端信息组织”，而不是欧美独立应用或 SaaS 营销页。

应坚持：

1. 信息密度适中：首屏要能看到关键事项、状态和入口，不追求大面积留白制造“高级感”。
2. 操作路径明确：首页、全部、我的三个主入口要符合国内用户对工具类 App 的直觉，不隐藏高频操作。
3. 卡片轻量聚合：卡片负责分组和承托内容，不做过厚阴影、过大圆角、过强渐变。
4. 色彩日常化：主题色应清透、低压力、偏生活服务感；避免过饱和荧光色、欧美潮牌式撞色和大面积黑底霓虹。
5. 动效服务反馈：保留按压、切换、进入、保存反馈等微动效，但避免欧美作品集式长动画、夸张弹跳和炫技转场。
6. 设置页本土化：`我的` 页应像国内常见 App 的设置中心/个人中心聚合页，分区清晰、入口轻、描述短，不做营销落地页式大 hero。
7. 商业化克制：会员、Pro、权益入口可以存在，但不要出现金币、皇冠、强促销角标、夸张价格锚点或“割裂工具感”的营销组件。

明确避免：

1. 欧美 SaaS landing page 风格：大标题、大副标题、大渐变 hero、大面积插画占首屏。
2. 海外效率 App 的极简空旷感：按钮太少、入口隐藏、需要用户探索才知道怎么用。
3. 国外社交/创意类 App 的强个性视觉：高饱和撞色、重噪点、霓虹玻璃、夸张 3D 图形。
4. 国内老式工具箱风格：蓝灰硬边框、密集按钮、全粗标题、传统表格化设置项。
5. 过度“年轻化”的贴纸化：大量表情、卡通贴纸、浮夸插画会削弱到期提醒的可靠感。

## 4. 字体与字号规范

默认不再使用“全局大字 + 全粗”。

推荐字号：

1. 页面标题：`24`，行高 `31`，字重 `700`
2. 卡片标题：`16`，行高 `22`，字重 `600`
3. 分区标题：`15`，行高 `21`，字重 `700`
4. 正文：`14`，行高 `21`，字重 `400`
5. 辅助文案：`13`，行高 `19`，字重 `400`
6. 标签/角标：`12`，行高 `16`，字重 `600`
7. 大数字：`22-24`，字重 `700`，仅用于首页总览等强数据场景

使用限制：

1. `900` 字重只允许用于少量图标 glyph 或特别短的数字，不用于普通标题和列表行。
2. 主要标题优先用 `700`，不要默认 `800/900`。
3. 同一张卡片内最多一个强强调文本。
4. 英文 eyebrow 不应大写加粗泛滥，能不用就不用。

## 5. 间距、圆角与尺寸规范

推荐尺寸：

1. 页面左右边距：`16`
2. 页面内容 gap：`14-16`
3. 卡片内边距：`14`
4. 列表行内边距：水平 `14`，垂直 `12`
5. 主按钮高度：`44`
6. 次级按钮高度：`38-40`
7. 筛选 chip 高度：`32-34`
8. 小图标容器：`34-38`
9. 普通卡片圆角：`16`
10. 大容器圆角：`20`
11. 胶囊圆角：`999`

使用限制：

1. `24+` 圆角只用于底部 Tab、特殊 hero 或浮层，不作为普通卡片默认值。
2. `20` 以上 padding 不作为普通页面默认值。
3. 行动按钮不要用过高 padding 伪装重要性，应通过颜色、图标和文案表达状态。

## 6. 色彩与主题规范

应用必须支持：

1. 浅色模式
2. 深色模式
3. 跟随系统
4. 主题色切换

首批主题色：

1. 青绿色：默认，清爽、生活化
2. 蓝色：更通用、更效率工具感
3. 黄色：更轻快，但必须降低饱和度，避免廉价警告感
4. 浅绿色：更柔和、更日常

色彩边界：

1. 主色只用于焦点、主按钮、激活态和少量强调。
2. 卡片背景应优先使用 surface，不要所有模块都铺主色 soft。
3. 状态色固定语义：逾期红、临期橙、完成绿。
4. 深色模式不是简单反色，需要降低边框对比，提高文字层级可读性。

主题色补充要求：

1. 默认青绿色应保持“生活服务 + 清爽可信”，不要做成海外健康 App 的高冷极简绿。
2. 蓝色主题应偏国内工具类通用蓝，避免金融后台感或欧美 SaaS 强商务蓝。
3. 黄色主题必须低饱和、偏暖柔，不能接近警告黄或促销黄。
4. 浅绿色主题应偏自然、轻松，但不要让可读性下降。
5. 后续如新增主题色，必须先说明其在中国大陆市场中的情绪定位和适用人群。

## 7. 我的页规范

`我的` 页应该像设置中心，而不是营销落地页。

结构建议：

1. 顶部使用轻量账户/应用状态卡，不使用过大的 hero 标题。
2. 会员、通知、主题外观属于高频设置区。
3. 数据备份、反馈、关于属于常用工具区。
4. 隐私、协议、权限说明属于合规说明区。
5. 列表行使用轻图标、小标题、短描述和轻箭头，不要所有标题加粗。

## 8. 后续执行要求

1. 新增页面必须优先使用 `src/theme/` 里的 token。
2. 不再直接新增散落的 `fontWeight: '900'`。
3. 不再在页面内随意写 `28/30` 页面标题。
4. 若需要特殊视觉效果，先说明它承担的信息层级，而不是为了“更醒目”。
5. 主题色、深浅色相关能力必须通过统一主题上下文读取，不要写死单一颜色。
6. 新增视觉方案必须先通过“中国大陆市场适配”检查：信息密度、入口显性、色彩克制、动效目的、商业化克制。
7. 引入海外 App 参考时必须说明“只参考什么、不照搬什么”，避免把本项目带偏为欧美 SaaS 或海外独立 App 风格。
